## 🏗️ Project Highlight — Telecom Service Provider (3-Year Architecture)

The architecture was designed by the architecture team; I was responsible for **end-to-end implementation**.

| Layer | Details |
|---|---|
| Frontend | Next.js — deployed on EC2 (private subnet) |
| Backend | Node.js — deployed on EC2 (private subnet) |
| Load Balancer | ALB in public subnet (public-facing) |
| Database | AWS RDS (MySQL) with Multi-AZ for high availability |
| Scaling | Auto Scaling Groups for EC2 |
| Environments | Dev, Stage, UAT, Prod — isolated via multi-VPC architecture |
| IaC | Terraform — VPC, subnets, EC2, ALB, Auto Scaling, RDS |
| SCM | GitHub — environment-based branching strategy |
| CI/CD | GitHub Actions |
| Monitoring | AWS CloudWatch (logs & services), Route53 Health Checks |
| Notifications | AWS SNS |
| Documentation | Confluence (centralized docs store) |
